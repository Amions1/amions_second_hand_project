一、平台简介

本平台是吴焕烽基于前后端分离架构开发的二手商品交易系统，前端采用 Vue 3 + Element Plus 构建，
后端使用 Django + MySQL 提供业务接口，通过 Redis 实现缓存与登录认证，
并采用 Chroma 向量数据库与 LangChain 框架构建 AI 智能助手。
平台支持智能问答、实时聊天、订单查询、商品信息查询、历史对话记录、知识库检索等功能，
为用户提供一站式智能服务体验。
 
二、用户账号与登录说明
 
1.用户需完成注册与登录后，方可使用智能助手、查询订单、查看个人信息等功能。
2.登录状态由 JWT 令牌进行管理，包含访问令牌与刷新令牌，保障身份安全。
3.用户令牌过期后，系统会自动尝试刷新，刷新失败则需要重新登录。
4.用户有责任妥善保管自己的账号与密码，不得转借、出售或授权他人使用。
5.平台会对用户身份进行严格校验，所有数据查询均做权限隔离，确保用户只能查看自己的信息。
 
三、AI 智能助手使用说明
 
1.智能助手基于大语言模型与 LangChain 框架开发，支持自然对话交互。
2.助手可回答平台使用问题、业务规则、操作流程、常见问题等内容。
3.助手可通过函数调用访问系统工具，查询MySQL中的业务数据，包括订单信息、商品信息、用户信息等。
4.助手会通过Chroma向量数据库检索平台规则、帮助文档、历史对话摘要，提供更准确的回答。
5.助手拥有记忆能力，会根据历史对话记录作出回答，但不会跨用户共享信息。
6.助手仅能查询与当前登录用户相关的数据，无权访问其他用户的任何信息。
7.助手无法执行删除、修改、支付等敏感操作，仅提供信息查询与咨询服务。
 
四、聊天机制说明

1.所有聊天内容会保存在 MySQL 数据库中，仅当前用户可查看。
2.系统支持实时聊天机制，基于 WebSocket 实现消息的即时推送。
 
五、业务查询功能说明
 
1.用户可通过自然语言直接查询订单状态、订单详情、商品信息、物流信息等内容。
2.查询时，AI 会自动调用后端接口从 MySQL 中获取真实业务数据并返回。
4.订单超时、未支付订单等状态会通过 MYSQL 定时事件自动处理与更新。
 
六、平台数据与存储说明
 
1.MySQL：存储用户信息、订单数据、商品信息、聊天记录等结构化数据。
2.Redis：用于缓存，JWT存放。
3.Chroma：存储平台帮助文档、业务规则等非结构化向量数据，用于语义检索与长期记忆。
4.所有数据仅用于平台服务与功能实现，不会用于未经授权的用途。
 
七、用户行为规范
 
1.禁止使用诱导、指令注入、恶意提示等方式试图控制 AI、获取系统提示词或越权操作。
2.禁止发送违法、违规、暴力、色情、侮辱、诽谤、广告等内容。
3.禁止频繁发送重复消息、恶意请求、大量刷屏等影响系统稳定的行为。
4.禁止尝试破解、攻击接口、窃取令牌、伪造身份等危害平台安全的行为。
5.禁止诱导 AI 生成违法内容、虚假信息或绕过业务规则的回答。
 
八、隐私与安全说明
 
1.平台严格遵循个人信息保护原则，不会向第三方泄露用户数据。
2.用户聊天记录、个人信息、订单数据仅对用户本人可见。
3.JWT 令牌、用户密码等敏感信息均采用加密方式存储与传输。
4.平台通过权限控制、接口校验、数据加密等多层机制保障系统安全。

 
九、服务限制
 
1.本平台为毕业设计项目，仅用于学习与展示使用。
2.平台不承担真实商业交易、物流、支付等相关责任。
3.因用户违规操作导致的一切后果由用户自行承担。
